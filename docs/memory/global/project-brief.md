# Project Brief

## Nome do Projeto
Forja (Base de Instruções Reutilizável)

## Escopo
Centralizar padrões operacionais para desenvolvimento solo e reutilizar essas instruções em outros projetos via links simbólicos.

## Objetivos
- Padronizar workflow de entrega com gitflow adaptado.
- Tornar memory-bank parte obrigatória do ciclo.
- Reduzir skills para uma allowlist mínima e funcional.
- Introduzir guard rails de testes por tarefa.

## Limites (O que NAO faz parte)
- Não implementar governança pesada de equipe.
- Não adotar ferramentas novas sem ganho claro.
- Não manter catálogo amplo de skills sem curadoria.

## Stakeholders
- Desenvolvedor principal (solo dev)
